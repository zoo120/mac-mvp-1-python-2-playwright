# 闲鱼热卖品监测 MVP 设计规格

## 目标与边界

构建一个仅在 Mac 本地运行的闲鱼选品辅助工具。用户通过关键词批量采集闲鱼搜索结果，系统将数据保存到 SQLite，并依据透明、可调整的规则生成候选品，最后通过 Streamlit 查看和筛选。

第一版不部署服务器，不提供登录与权限系统，不执行自动私信、下单、评论，也不尝试绕过闲鱼登录、验证码或其他平台安全措施。

## 技术方案

- Python 3.10 及以上。
- Playwright 驱动可见的 Chromium 浏览器。
- 独立持久化浏览器目录保存闲鱼登录会话；需要登录或验证时允许用户手动完成。
- SQLite 保存关键词、采集快照和候选品。
- Streamlit 提供四个本地管理页面。
- pytest 覆盖数据库、解析与候选规则的核心行为。

采集器作为独立命令行程序运行，Streamlit 只负责数据展示和关键词启停。这样浏览器长任务不会被 Streamlit 刷新或重启打断。

## 文件职责

- `requirements.txt`：运行与测试依赖。
- `README.md`：安装、初始化、采集、登录和启动后台说明。
- `init_db.py`：创建表、索引并幂等写入内置关键词。
- `database.py`：数据库连接、查询和写入接口。
- `rules.py`：过滤词、加权词和候选品判定。
- `crawler.py`：浏览器会话、页面解析、节流、错误记录和采集主流程。
- `app.py`：四个 Streamlit 页面。
- `tests/`：数据库、规则和 HTML 解析测试。

## 数据模型

### keywords

字段为 `id, keyword, category, enabled, priority, note, created_at, updated_at`。`keyword` 唯一；重复初始化时保留已有的启停、优先级和备注。

首批关键词使用用户给出的完整列表。第一版根据用途设置宽泛分类，如遮阳户外、餐饮设备、学习家具、体育器材、通风制冷、商用厨房、宠物设备、农业机械和家具；用户之后可以直接在数据库或后台扩展。

### crawled_items

字段为 `id, keyword, title, price, want_count, location, seller_type, item_url, image_url, raw_text, crawl_date, created_at`。

同一 `keyword + item_url + crawl_date` 视为同一日快照，重复运行时更新已有行。无法获得链接时不做该唯一约束，仍保存结果，避免丢数据。价格保存为数值，无法解析时为空；想要数保存为整数，无法解析时为空。

### product_candidates

字段为 `id, product_name, keyword, title, price, want_count, item_url, reason, risk_level, recommendation_status, created_at`。

同一采集结果每次重新判定时，以 `keyword + item_url` 更新当前候选记录；无链接结果以新记录保存。

## 采集流程

1. 初始化数据库，读取 `enabled = 1` 的关键词，按 `priority` 降序、`id` 升序处理。
2. 使用 `.playwright-profile` 启动可见持久化 Chromium。
3. 对每个关键词打开闲鱼搜索页，等待结果出现；若页面提示登录或验证，在终端提示用户手动处理并按回车继续。
4. 从多组候选选择器中选择第一个可用结构，最多提取前 20 条。
5. 每条结果先尝试结构化选择器，再从卡片全文中兜底提取价格、想要数、地点和卖家类型。任何单字段失败均保存为空。
6. 保存采集快照，立即运行候选规则并写入候选表。
7. 单个商品或关键词失败时写入 `logs/crawler.log`，继续后续任务。
8. 每两个关键词之间随机等待 10–30 秒；最后一个关键词之后不等待。

命令行支持可选 `--keyword` 仅采集一个已存在关键词，便于首次验证；默认采集全部启用关键词。还支持 `--limit`，默认值和最大值均为 20。

## 页面解析容错

闲鱼页面结构可能变化，因此采集器不依赖单一类名：

- 商品卡片使用若干语义与链接模式候选选择器。
- 标题、图片、价格分别尝试属性、子元素和全文解析。
- 商品链接统一转为绝对 URL。
- 选择器全部失效时记录清晰警告，不让整批任务崩溃。
- 保存 `raw_text` 以便页面变化后回溯和改进解析。

第一版不保证未登录状态下一定能获得 20 条，也不保证页面未展示的字段可以采集。

## 候选品规则

过滤词和加权词使用用户给出的完整词表，匹配时忽略英文大小写。

规则优先级从高到低：

1. 命中过滤词：`recommendation_status = 不建议`，`risk_level = 高`。
2. 明显低价小百货：价格不高于 30 元，且未命中加权词：`不建议`，风险为中。
3. 命中加权词且想要数不低于 20：`可交付候选`，风险为低。
4. 命中加权词且价格不低于 200 元：`可观察`，风险为低。
5. 其他结果：`可观察`，风险为中。

`reason` 保存实际命中的词、价格和想要数依据，确保判断可解释。`product_name` 第一版直接使用清洗后的标题。

阈值集中定义在 `rules.py`，便于下一版根据真实数据调整。

## Streamlit 页面

### 今日概览

显示当天采集记录数、启用关键词数、当天生成或更新的候选品数，并提供最近采集结果预览。

### 关键词管理

表格展示关键词、分类、启用状态、优先级和备注。支持对单个关键词启用或停用；第一版不实现新增、删除和批量编辑。

### 采集结果

支持按关键词、价格区间和最低想要数筛选，结果按采集日期和创建时间倒序展示。空价格或空想要数只在没有对应数值过滤时显示。

### 候选品

支持按关键词、推荐状态和风险等级筛选，展示标题、价格、想要数、原因及可点击链接。Streamlit 表格中的标题和链接可由用户选中复制。

## 错误处理与合规节流

- 数据库写入按单个关键词事务处理，失败时回滚该关键词并继续其他关键词。
- 日志同时写文件和终端，包含时间、关键词和错误摘要。
- 默认可见浏览器，用户可以自然处理登录和验证。
- 关键词之间强制随机等待 10–30 秒，不提供小于 10 秒的命令行参数。
- 不实现并发采集、代理池、验证码绕过或其他规避平台控制的能力。

## 验收标准

- `python init_db.py` 可重复执行，三张表和内置关键词存在且不重复。
- `python crawler.py --keyword 遮阳棚 --limit 3` 能打开浏览器；页面可访问时保存不超过 3 条，页面变化或登录受限时记录错误并正常退出。
- 候选规则对过滤词、加权高价、加权高想要数和低价小百货给出预期状态。
- `streamlit run app.py` 可打开四个页面，并可启停关键词及筛选数据。
- 自动化测试无需访问闲鱼网络即可运行。

## 已知限制

- 闲鱼页面、风控策略及可见字段会变化，真实采集效果需在用户当前网络和账号环境中验证。
- 第一版没有定时调度、数据导出、趋势计算、商品去重聚类或图片识别。
- Git 提交依赖本机 Apple 命令行开发工具；若未安装，文档和代码仍可生成，但无法在当前环境完成提交。
