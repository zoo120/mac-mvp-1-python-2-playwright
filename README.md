# 闲鱼选品助手

一个闲鱼选品辅助工具。默认打开后是“学员傻瓜模式”：输入商品词，点搜索，看热度结果，一键保存文案和图片。管理员可以打开侧边栏开关，查看采集结果、候选品、关键词等高级功能。

当前支持两种运行方式：

- 本地版：在你的 Mac 上运行，地址是 `http://localhost:8501`。
- 线上部署版：通过 Docker 放到公网服务器，让全国学员打开同一个网址使用。部署说明见 `ONLINE_DEPLOY.md`。

> 本工具只做低频数据采集与人工选品辅助，不自动私信、下单或评论，也不绕过登录、验证码和平台安全验证。请遵守闲鱼规则及适用法律，并根据页面实际情况控制使用频率。

## 运行环境

- macOS
- Python 3.10 或更高版本
- 首次安装依赖和 Chromium 时需要网络
- 采集时会打开一个可见浏览器窗口

若终端执行 `python3 --version` 只提示安装 Apple 开发工具，可以先从 [Python 官网](https://www.python.org/downloads/macos/) 安装 Python，然后重新打开终端。

## 1. 安装依赖

在项目目录打开终端：

```bash
python3 -m venv .venv
source .venv/bin/activate
python -m pip install --upgrade pip
pip install -r requirements.txt
playwright install chromium
```

以后每次重新打开终端，只需进入项目目录并激活环境：

```bash
source .venv/bin/activate
```

## 2. 初始化数据库

```bash
python init_db.py
```

这个命令会在项目目录创建 `xianyu_monitor.db`，建立本地数据表并写入 30 个内置关键词。它可以重复执行，不会重复添加关键词，也不会覆盖你已经调整过的启停状态。

## 3. 运行采集

建议第一次只测试一个关键词和少量商品：

```bash
python crawler.py --keyword 遮阳棚 --limit 3
```

确认运行正常后，采集全部已启用关键词：

```bash
python crawler.py
```

每个关键词默认最多采集 20 条。关键词之间强制随机等待 10–30 秒，全部关键词因此需要较长时间，这是预期行为。也可以指定其他数据库：

```bash
python crawler.py --db data/my-monitor.db --limit 20
```

### 首次登录

采集器使用项目内的 `.playwright-profile/` 保存独立浏览器会话。首次遇到闲鱼登录或安全验证时：

1. 在打开的 Chromium 窗口中手动登录或完成验证。
2. 回到终端按回车。
3. 后续运行会尽量复用这个会话。

不要把 `.playwright-profile/` 分享给他人，其中可能包含登录状态。不要让普通 Chrome 和采集器共用同一个用户数据目录。

## 4. 启动本地网页工具

最简单方式：双击项目里的这个文件：

```text
启动闲鱼选品助手.command
```

双击后会弹出终端窗口，看到 `Local URL: http://localhost:8501` 就是启动成功。这个终端窗口不要关闭。

也可以手动在终端运行：

```bash
streamlit run app.py
```

终端会显示本地地址，通常是 `http://localhost:8501`。

默认页面就是给学员用的“闲鱼选品助手”。侧边栏默认不会展开复杂后台；你自己需要管理数据时，再打开“显示管理员功能”。

如果学员和你在同一个 Wi‑Fi，页面侧边栏会显示“同 Wi‑Fi 学员打开”的地址，例如：

```text
http://192.168.1.8:8501
```

学员在自己电脑或手机浏览器输入这个地址，可以访问你的这台电脑上的工具。注意：这只适合同一个 Wi‑Fi；不在同一个网络时打不开。全国学员使用要走“线上部署版”，见 `ONLINE_DEPLOY.md`。

管理员功能包含：

- 今日概览：今日采集量、启用关键词数、今日候选品数。
- 关键词管理：启用或停用关键词。
- 采集结果：按关键词、价格和想要数筛选。
- 候选品：按关键词、推荐状态和风险筛选，复制标题并打开商品链接。
- 素材保存：从候选品里选一条商品，一键打开详情页并保存文案、商品信息和图片。

停止后台或采集器时，在对应终端按 `Control + C`。

## 5. 学员选品助手

学员只需要按页面提示做四步：

1. 输入商品关键词，例如 `床垫`。
2. 点击“第 2 步：开始搜索”。
3. 查看系统按“想要数”排序出来的结果。
4. 点击“保存文案和图片”。

保存成功后，页面会显示素材文件夹路径，并提供“在访达打开素材文件夹”按钮。文件夹里：

- `文案.txt`：商品详情页文案。
- `images`：商品图片。
- `商品信息.json`：商品链接、标题、图片链接等记录。

注意：闲鱼搜索页通常不直接展示真实销量，所以第一版用“想要数”作为热度参考，不等于真实销量。

## 6. 一键保存商品素材

后台进入“素材保存”页面后：

1. 在下拉框选择一个候选商品。
2. 点击“一键保存文案和图片”。
3. 如果弹出的 Chromium 要求登录或验证，先在浏览器里手动完成，然后回到后台再点一次保存。

保存成功后，素材会放在项目目录的 `saved_products/` 文件夹，每个商品一个独立文件夹，里面通常包含：

- `文案.txt`：详情页提取到的商品文案。
- `商品信息.json`：商品标题、链接、图片链接和保存时间。
- `images/`：下载到本地的商品图片。

这个功能是人工点选单条保存，不会批量高频打开详情页。

## 7. 线上部署版

项目已经加入线上部署文件：

- `Dockerfile`
- `render.yaml`
- `.dockerignore`
- `ONLINE_DEPLOY.md`

线上部署后，学员不需要安装 Python，也不需要打开终端，只需要访问公网网址。

但要注意：闲鱼可能要求登录或安全验证。服务器上通常没有你电脑这种可见浏览器窗口，所以第一版线上 MVP 可能会遇到风控失败。后续要做成稳定商用版，建议再加任务队列、账号登录后台、用户隔离和云存储。

## 数据与规则

本地文件：

- `xianyu_monitor.db`：SQLite 数据库。
- `logs/crawler.log`：采集日志；单商品或单关键词失败会记录在这里。
- `.playwright-profile/`：独立浏览器登录会话。
- `saved_products/`：一键保存的商品文案和图片素材。

主要业务表分别为：

- `keywords`：关键词、分类、启停、优先级和备注。
- `crawled_items`：每天的搜索结果快照与页面原文 `raw_text`。
- `product_candidates`：候选判断、原因、风险和推荐状态。
- `saved_product_assets`：已保存素材包的标题、链接、本地文件夹和图片数量。

第一版规则位于 `rules.py`：

- 命中过滤词：不建议。
- 未命中加权词且价格不高于 30 元：按低价小百货处理，不建议。
- 命中加权词且想要数不低于 20：可交付候选。
- 命中加权词且价格不低于 200 元：可观察。
- 其他结果：可观察，等待人工判断。

规则只是初筛，不代表销量、利润、合规或供货能力已经确认。交付给学员前仍需人工检查商品真实性、运费、供应稳定性和售后风险。

## 常见问题

### 浏览器能打开，但没有采集结果

闲鱼页面结构可能已经改变，或当前页面只对登录用户展示结果。先检查：

1. 浏览器中是否正常看到商品卡片。
2. 是否出现登录、验证码或安全验证。
3. `logs/crawler.log` 是否出现“未找到商品卡片”或选择器错误。

采集器保留了 `raw_text`，可用于后续调整解析规则。页面字段没有展示时，系统会保存为空，不会让整批采集崩溃。

### Chromium 尚未安装

重新运行：

```bash
playwright install chromium
```

### 数据库被占用

先停止其他正在运行的采集器。不要同时启动两个采集进程；持久化浏览器目录和 SQLite 都按单进程使用设计。

### 只想重新初始化关键词

直接再次运行：

```bash
python init_db.py
```

现有关键词的启停、优先级和备注不会被内置默认值覆盖。

## 运行测试

```bash
python -m pytest tests -q
```

测试不访问闲鱼网络，主要覆盖数据库幂等写入、规则优先级、字段解析、后台查询和交付清单。

## 第一版限制

- 已提供 Docker 线上部署基础配置，但不包含云平台账号、域名、登录账号池和复杂多用户权限。
- 不提供自动私信、下单、评论或验证码绕过。
- 不做并发或高频采集。
- 页面未展示的想要数、地点或卖家类型会保存为空。
- 闲鱼页面结构和风控策略随时可能变化，真实采集效果需要在当前账号和网络环境中验证。
